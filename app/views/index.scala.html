Scapig is an Open Source API Gateway with an integrated Developer Hub.

Third party developers can integrate with your APIs by registering an application and subscribing to APIs. Features provided by the Scapig platform include:

  • Oauth 2.0 Integration
  • Rate limiting
  • Scalable
  • Pluggable
  • Open Source
  • Active/Active architecture

Oauth 2.0 Integration

Scapig complies with the Oauth 2.0 RFC specifications. The authorization_code flow has been implemented and the other flows are to follow soon

Rate Limiting

You can rate limit applications accessing your APIs with the following configurable tiers:

  • Gold
  • Silver
  • Bronze

Scalable

Scapig has a microservice architecture. This allows you to selectively increase the capacity of the platform which is in more strain. As an example, you will need more instances of scapig-gateways or scapig-delegated-authority which are called on every request than on scapig-developer-hub which is hosting the developer hub.

Pluggable

Scapig is based on a series of microservices communicating through REST HTTP requests. It is fairly easy to replace one of its component by a bespoke component providing the REST contract is respected. As an example, the default scapig-developer-hub could be replaced by a frontend with a branded UI.

Open Source

Scapig is Open Source in Github

Active/Active Architecture

Scapig data is stored in MongoDB which has great support on replication over multiple datacenters.

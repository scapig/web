GET STARTED - CREATE AN APPLICATION

You can see an example of the developer hub at http://developer.scapig.com/
You will need to register to be able to create an application.
The application contains:
I. Application Details
1. Name: This will be shown to the end user when they authorize the application to access their data.
2. Description: Optional. An internal field to describe your application
3. Redirect URIs: The whitelist of URIs which are allowed when redirecting the user to the application server for the authorization_code flow.
4. Rate limit tiers: The level of rate limiting required for the application. It can be one of GOLD, SILVER or BRONZE.
II. Subscriptions
This contains the list of APIs on the platform and allow the developer to subscribe to the API.
III. Sandbox and Production Credentials
1. Client ID: This is the Oauth client_id of your application
2. Client Secret: This is the Oauth client_secret of your application
3. Server Token: This is the Oauth server_token of your application. You can access Application Restricted endpoints with this token.
When using Sandbox credentials, the API call will be routed to the sandbox endpoint (starting with /sandbox).
When using Production credentials, the API call will be routed to the live endpoint.

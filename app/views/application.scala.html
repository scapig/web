When a developer wants to integrate with your secure APIs, he will need to create an application. The application will contain the following:
Field Description
Redirect URI This is the uri the user will be redirected to once he completes his Oauth 2.0 journey
Sandbox Credentials Those are the client_id, client_secret and server_token of the Sandbox environment. When using those, the API Gateway will proxy the user to the Sandbox endpoint.
Production Credentials Those are the client_id, client_secret and server_token of the Production environment. When using those, the API Gateway will proxy the user to the Production endpoint.
Rate Limit Tiers The application will be given a rate limit tier. This will determine how many requests per minute and application can hit the API Gateway The possible values are:
  • BRONZE
  • SILVER
  • GOLD
